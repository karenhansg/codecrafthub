# CodeCraftHub

A lightweight Spring Boot REST API for managing courses. Data is persisted in a local JSON file (`courses.json`) using Jackson—no database required. Ideal for learning Spring Boot, REST design, and file-based storage.

## Features

- **Full CRUD API** — Create, read, update, and delete courses via HTTP endpoints
- **JSON file storage** — All courses are saved to `courses.json` in the project working directory
- **Auto-generated fields** — `id` (starting at 1) and `created_at` are set automatically on create
- **Input validation** — Required fields and allowed status values are enforced with clear error messages
- **Centralized error handling** — Consistent JSON error responses for validation, not-found, and file I/O failures
- **Beginner-friendly code** — Commented service, controller, and model classes

## Prerequisites

Before you begin, install:

| Requirement | Version |
|-------------|---------|
| Java JDK    | 17 or later |
| Apache Maven | 3.6 or later |

Verify your setup:

```bash
java -version
mvn -version
```

## Installation

1. **Clone or download the project**

   ```bash
   git clone <repository-url>
   cd codecrafthub
   ```

2. **Install dependencies and compile**

   ```bash
   mvn clean install
   ```

   Maven downloads Spring Boot and other dependencies on the first run. This may take a minute.

3. **Confirm the build**

   A successful build ends with `BUILD SUCCESS`. No separate database setup is needed.

## Running the Application

Start the server with Maven:

```bash
mvn spring-boot:run
```

Or run the packaged JAR after building:

```bash
java -jar target/codecrafthub-1.0.0.jar
```

The API is available at:

```
http://localhost:8080
```

On first startup, the app creates `courses.json` in the current working directory (usually the project root) with an empty array `[]` if the file does not exist.

To stop the server, press `Ctrl+C` in the terminal.

## Course Data Model

Each course has the following JSON fields:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `id` | number | Auto | Generated by the server, starting at 1 |
| `name` | string | Yes | Course title |
| `description` | string | Yes | Course summary |
| `target_date` | string | Yes | Format: `YYYY-MM-DD` |
| `status` | string | Yes | `"Not Started"`, `"In Progress"`, or `"Completed"` |
| `created_at` | string | Auto | ISO-8601 timestamp set on create; preserved on update |

Example response:

```json
{
  "id": 1,
  "name": "Spring Boot Basics",
  "description": "Learn REST APIs with Spring Boot",
  "target_date": "2026-12-31",
  "status": "Not Started",
  "created_at": "2026-06-28T14:30:00"
}
```

## API Documentation

Base URL: `http://localhost:8080/api/courses`

All request and response bodies use JSON. Set the header:

```
Content-Type: application/json
```

### Create a course

**`POST /api/courses`**

Creates a new course. Do not send `id` or `created_at`; they are generated by the server.

**Request body:**

```json
{
  "name": "Spring Boot Basics",
  "description": "Learn REST APIs with Spring Boot",
  "target_date": "2026-12-31",
  "status": "Not Started"
}
```

**Example:**

```bash
curl -X POST http://localhost:8080/api/courses \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Spring Boot Basics",
    "description": "Learn REST APIs with Spring Boot",
    "target_date": "2026-12-31",
    "status": "Not Started"
  }'
```

**Response:** `201 Created` with the created course object.

---

### Get all courses

**`GET /api/courses`**

Returns every course in `courses.json`.

**Example:**

```bash
curl http://localhost:8080/api/courses
```

**Response:** `200 OK` with a JSON array of courses (empty array `[]` if none exist).

---

### Get a course by ID

**`GET /api/courses/{id}`**

Returns a single course.

**Example:**

```bash
curl http://localhost:8080/api/courses/1
```

**Response:** `200 OK` with the course object, or `404 Not Found` if the id does not exist.

---

### Update a course

**`PUT /api/courses/{id}`**

Replaces the editable fields of an existing course. The original `created_at` value is kept.

**Request body:**

```json
{
  "name": "Spring Boot Basics",
  "description": "Updated course description",
  "target_date": "2026-12-31",
  "status": "In Progress"
}
```

**Example:**

```bash
curl -X PUT http://localhost:8080/api/courses/1 \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Spring Boot Basics",
    "description": "Updated course description",
    "target_date": "2026-12-31",
    "status": "In Progress"
  }'
```

**Response:** `200 OK` with the updated course, or `404 Not Found` if the id does not exist.

---

### Delete a course

**`DELETE /api/courses/{id}`**

Removes a course from `courses.json`.

**Example:**

```bash
curl -X DELETE http://localhost:8080/api/courses/1
```

**Response:** `204 No Content` on success, or `404 Not Found` if the id does not exist.

---

### Error responses

Failed requests return JSON in this shape:

```json
{
  "timestamp": "2026-06-28T14:30:00",
  "status": 400,
  "error": "Bad Request",
  "message": "Field 'name' is required and cannot be blank."
}
```

| HTTP Status | When it occurs |
|-------------|----------------|
| `400 Bad Request` | Missing required fields, invalid status, or malformed JSON |
| `404 Not Found` | Course id does not exist |
| `500 Internal Server Error` | Failed to read or write `courses.json` |

## Troubleshooting

### `java: command not found` or wrong Java version

Install JDK 17 or later and ensure it is on your `PATH`:

```bash
java -version
```

On macOS with Homebrew:

```bash
brew install openjdk@17
```

### `mvn: command not found`

Install Maven and confirm it is available:

```bash
mvn -version
```

On macOS with Homebrew:

```bash
brew install maven
```

### Port 8080 already in use

Another process is using port 8080. Either stop that process or change the port in `src/main/resources/application.properties`:

```properties
server.port=8081
```

Then restart the app and use `http://localhost:8081` in your requests.

### `Connection refused` when calling the API

- Confirm the application started successfully (look for `Started CodeCraftHubApplication` in the logs).
- Use the correct base URL: `http://localhost:8080/api/courses`.
- Ensure the server is still running in your terminal.

### `400 Bad Request` — missing or invalid fields

- Include all required fields: `name`, `description`, `target_date`, `status`.
- Use `target_date` in `YYYY-MM-DD` format (e.g. `"2026-12-31"`).
- Set `status` to exactly one of: `"Not Started"`, `"In Progress"`, `"Completed"` (case and spacing matter).

### `404 Not Found` when fetching or updating a course

The course id does not exist. List all courses to see valid ids:

```bash
curl http://localhost:8080/api/courses
```

### `500 Internal Server Error` — file read/write failure

- Check that the app has write permission in the directory where it runs (typically the project root).
- Ensure `courses.json` is valid JSON (an array of course objects). If the file is corrupted, back it up and replace it with `[]`.
- Do not edit `courses.json` while the server is running unless you understand the risk of concurrent writes.

### Changes to `courses.json` are not visible

The file is read from the **working directory** where you start the app. If you run Maven from a different folder, `courses.json` may be created elsewhere. Always start the app from the project root:

```bash
cd codecrafthub
mvn spring-boot:run
```

### Build fails with dependency download errors

- Check your internet connection.
- Retry: `mvn clean install -U`
- If behind a corporate proxy, configure Maven settings in `~/.m2/settings.xml`.
